# PR 본문 템플릿

`create-pr` 스킬이 PR 본문을 작성할 때 따르는 구조. diff/커밋에서 근거를 찾을 수 없는 섹션은 빈 칸으로 채우지 말고 **섹션째로 생략**한다. `Summary`만 항상 포함한다.

---

## Summary

- {가장 중요한 변경 1}
- {가장 중요한 변경 2}
- {가장 중요한 변경 3, 있다면}

## Why

{이 변경이 필요했던 이유나 배경. 커밋 메시지·이슈 언급에서 근거를 찾을 수 있을 때만 작성한다. 없으면 생략한다.}

## Changes

- {영역/파일별 변경 요약}

변경 파일이 3개 이하이고 `Summary`만으로 충분히 설명되면 이 섹션은 생략한다.

## Test plan

- [ ] {실행한 테스트/빌드/린트 명령과 결과, 예: `bun run test` 통과}
- [ ] {수동으로 확인한 시나리오}

## Notes

{브레이킹 체인지, 후속 작업, 리뷰어가 특별히 봐야 할 부분. 없으면 생략한다.}
